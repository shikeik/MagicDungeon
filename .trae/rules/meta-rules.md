# 规则编写范式 (Rule of Rules)

为确保 AI 快速理解并执行规则，所有 `.trae/rules` 下的规则文档必须遵循以下范式：

## 1. 核心原则
*   **言简意赅**: 避免废话、修辞和背景介绍，直接陈述行动指令。
*   **原子化**: 每个规则文件应只描述单一环节或概念，便于组合复用。
*   **无歧义**: 使用“必须”、“严禁”、“可选”等强约束词。

## 2. 格式要求
*   **标题**: 清晰简短，不带“阶段X”等流程绑定词 (如 `任务启动` 而非 `第一阶段：任务启动`)。
*   **引用**: 使用相对路径引用其他规则文件。
*   **流程图**: **必须使用紧凑 ASCII 文本图**。
    *   允许使用简单的箭头和分支符号 (↓, →, ├─, └─)。
    *   避免过度复杂的 Mermaid 图表。
    *   **推荐范式**:
    ```text
    Start -> Process A -> Process B
      ↓
    [Check] Is OK?
      ├─ [No] -> (Back to Start)
      └─ [Yes] -> End
    ```

## 3. 内容结构
1.  **触发条件/前置检查**: 何时执行此规则。
2.  **核心动作**: 具体的步骤列表 (1, 2, 3...)。
3.  **输出/跳转**: 执行后的产物或下一步指向。
