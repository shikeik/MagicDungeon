# 第四阶段：交付与版本 (Deliver)

1.  **版本决策** (必须):
    - **内容性更新**: 必须执行发版。
        - **常规修复/优化**: 执行 `gradle bumpPatch`。
        - **次版本特性**: 执行 `gradle bumpMinor` (需遵循次版本流程: 确定主题 -> 开发 -> 提版 -> 下个主题 -> 提交)。
    - **纯杂项/文档**: 可跳过发版。
2.  **规范提交** (必须):
    - 使用中文编写 `commit_msg.txt`。
    - 执行 `git commit -F commit_msg.txt`。
    - **禁止 Push**: 仅本地提交。
