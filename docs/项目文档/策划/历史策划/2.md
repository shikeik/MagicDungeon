# 移植现状评估与行动计划

## 1. 现状评估
经过代码审查，确认 H5 版本的核心逻辑已高度还原到 GDX 项目中：
- **核心架构**: `NewDungeonGame` 已建立，作为新的游戏入口。
- **游戏循环**: `GameScreen` 实现了完整的更新(`update`)与渲染(`render`)循环，包含视锥体剔除优化。
- **业务逻辑**: 地图生成(`Dungeon`)、实体行为(`Player`, `Monster`)、物品系统均已移植。
- **美术资源**: `SpriteGenerator` 成功使用 `Pixmap` 复刻了原版 Canvas 绘图逻辑，无需外部素材。

**发现的问题**:
- **启动入口未更新**: `Lwjgl3Launcher.java` 目前仍指向旧的 `GdxLauncher`，这会导致运行项目时无法进入新移植的游戏内容。

## 2. 行动方案

### 第一步：修正启动配置 (立即执行)
- 修改 `lwjgl3/src/main/java/com.goldsprite.magicdungeon2/lwjgl3/Lwjgl3Launcher.java`。
- 将 `new GdxLauncher()` 替换为 `new com.goldsprite.magicdungeon2.core.MagicDungeon2Game()`。

### 第二步：功能验证
- 启动游戏，验证以下核心路径：
    1. **移动**: WASD 控制角色移动，检查碰撞检测。
    2. **战斗**: 靠近怪物自动攻击，检查扣血逻辑。
    3. **交互**: 拾取药水/装备，按 E 下楼。
    4. **UI**: 检查 HUD 显示（血量、日志）是否正常。

### 第三步：后续优化 (Backlog)
- **输入手感**: 检查移动是否存在过快或粘滞问题，必要时引入输入缓冲。
- **UI 美化**: 使用 `VisUI` 替换临时的 `GameHUD` 实现。
- **代码清理**: 移除 `examples` 包下不再使用的旧演示代码。
