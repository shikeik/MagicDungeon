## 四、可靠 UDP 传输层 (ReliableUdpTransport)

### 4.1 设计思路

现有 `UdpSocketTransport` 只提供 Best-Effort 投递。状态同步 (StateSync) 每帧发送不需要可靠性，但 Spawn/Despawn/RPC 等一次性关键消息丢包会导致客户端状态不一致。

**解决方案**: 装饰器模式 — `ReliableUdpTransport` 包装 `UdpSocketTransport`，实现同一个 `Transport` 接口，对上层 `NetworkManager` 完全透明。

### 4.2 双通道路由

根据 packetType 自动路由到不同通道：

| packetType | 名称 | 通道 | 原因 |
|---|---|---|---|
| `0x10` | StateSync | **不可靠** | 每帧发送，丢了不影响 |
| `0x11` | Spawn | **可靠** | 丢包 = 实体缺失 |
| `0x12` | Despawn | **可靠** | 丢包 = 实体残留 |
| `0x20` | ServerRpc | **可靠** | 丢包 = 操作丢失 |
| `0x21` | ClientRpc | **可靠** | 丢包 = 回复丢失 |

### 4.3 数据包格式

```
可靠包: [0x01][seqNum:2B][ackNum:2B][payload:NB]
不可靠: [0x00][0x00][0x00][0x00][0x00][payload:NB]  (5字节头占位)
ACK包:  [0x02][0x00][0x00][ackNum:2B]                (无 payload)
```

- **seqNum**: 16 位循环序列号 (0~65535)，发送方单调递增
- **ackNum**: 接收方已成功收到的最新序列号
- 序列号新旧比较使用环形算法: `(a - b) & 0xFFFF < 32768`

### 4.4 重传机制

```
发送可靠包 → 加入 PendingBuffer(seqNum, data, timestamp)
               ↓
每帧 tickReliable():
  遍历 PendingBuffer:
    if (now - timestamp > 200ms):
      if (retries >= 5): 丢弃, 打印警告
      else: 重新发送, retries++, 更新 timestamp
               ↓
收到 ACK(ackNum):
  从 PendingBuffer 移除 seqNum <= ackNum 的所有包
```

### 4.5 接收端去重

```
维护 expectedSeqNum (期望收到的下一个序列号)
收到可靠包(seqNum):
  if seqNum < expectedSeqNum: 丢弃 (重复/旧包)
  else: expectedSeqNum = seqNum + 1, 投递给上层, 发送 ACK
```

## 五、地图墙体 & 边界系统

### 5.1 地图参数

| 参数 | 值 | 说明 |
|---|---|---|
| MAP_WIDTH | 2000px | 地图宽度 |
| MAP_HEIGHT | 1500px | 地图高度 |
| 墙体数量 | 5~10 | 随机生成 |
| 墙体尺寸 | W: 60~200, H: 20~100 | 单个墙体 |
| 墙体边距 | 100px | 距地图边界最小距离 |
| 边界线宽 | 3px | 边界线框线宽 |
| 墙体线宽 | 2px | 墙体线框线宽 |

### 5.2 确定性同步

双端使用相同种子生成完全一致的墙体布局，无需额外网络同步：

```java
long mapSeed = roomName.hashCode(); // 双端一致
MapCollisionUtils.generateWalls(mapSeed, MAP_WIDTH, MAP_HEIGHT);
```

### 5.3 碰撞系统

#### 坦克碰撞
1. **边界限制**: 每帧将坦克坐标 Clamp 到 `[halfSize, mapSize-halfSize]`
2. **墙体推回**: AABB 重叠检测 → 取四方向最小穿透深度 → 推回

#### 子弹碰撞
- 子弹 AABB (8x8) 与墙体 AABB 碰撞 → 命中墙体时销毁子弹
- 子弹超出 MAP_WIDTH/MAP_HEIGHT 时销毁

### 5.4 渲染

```java
// 地图边界 — 较粗线框
neon.drawRect(0, 0, MAP_WIDTH, MAP_HEIGHT, 0, 3f, BOUNDARY_COLOR, false);

// 墙体 — 半透明填充 + 线框
neon.drawRect(x, y, w, h, 0, 0, Color(0.2, 0.3, 0.5, 0.3), true);  // 填充
neon.drawRect(x, y, w, h, 0, 2f, WALL_COLOR, false);                 // 线框
```

### 5.5 工具类提取 (MapCollisionUtils)

将碰撞逻辑从 `NetcodeTankOnlineScreen` 提取为独立静态工具类，便于单元测试：

| 方法 | 功能 |
|---|---|
| `rectOverlap(ax,ay,aw,ah, bx,by,bw,bh)` | AABB 矩形重叠检测 |
| `clampToBoundary(x,y,halfSize,mapW,mapH)` | 坐标边界 Clamp |
| `generateWalls(seed,mapW,mapH)` | 确定性墙体生成 |
