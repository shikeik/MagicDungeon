# v0.7.1 Supabase 云大厅实装与重构报表

## 1. 概述与核心目标

在本次 v0.7.1 的重大迭代中，我们成功并完整地将 `SupabaseLobbyScreen` 及其配套的底层网络 API（`RoomManager`, `SupabaseClient`）实装到项目中，从而彻底代替了传统的局域网或固定 IP 直连模式。
该方案采用 **Serverless BaaS (Supabase) 作为 UDP P2P 联机的“信令大厅”**，使所有客户端能够在无需自己搭建中央服务器的情况下，通过公网相互发现并建立连接。

此次更新不仅仅是简单的功能堆砌，更是对整体架构的深度优化。我们完成了依赖配置的全局化剥离、解决了极其隐蔽的跨时区同步 Bug，并将云大厅模块成功整合进项目的核心测试入口中。

---

## 2. 核心系统实现与技术重构

### 2.1 底层数据通讯 (Data Layer)

- **SupabaseClient**: 核心网络通讯类。封装了 LibGDX 的 `Net.HttpRequest` 请求，使用基于 REST API 的标准 `POST`, `GET`, `PATCH`, `DELETE` 与我们的 Supabase 云端 `rooms` 表进行 JSON 数据交互。
- **RoomModel**: 纯 POJO 实体类。直接映射云端表结构，负责房间的状态序列化和反序列化。包含以下字段：`id`, `host_ip`, `host_port`, `room_name`, `current_players`, `max_players`, `status`, `last_ping`。

### 2.2 全局配置架构 (Configuration Layer) **[New]**

- **SupabaseConfig 统一定义**: 彻底废除了原先散落在多个 UI 和逻辑类中的硬编码 URL 和 Key。新增了专用的全局常量类 `SupabaseConfig.java` 控制所有 Supabase 的接入凭证（`URL` 与 `PUBLISHABLE_KEY`）。这赋予了系统更高的安全性、可变性以及更低的代码维护成本。
- **依赖注入重构**: `RoomManager` 及其消费者（如 `SupabaseLobbyScreen`、`SupabaseClientTest` 测试类）已被全面改造为从 `SupabaseConfig` 中动态读取安全凭证。

### 2.3 IP 解析与心跳维系 (Networking Layer)

- **PublicIPResolver**: 为了解决公网 P2P 打洞需求首要的公网 IP 嗅探问题，集成了 `https://checkip.amazonaws.com` 接口。能在建房前异步获取主机的公网 IPv4。
- **RoomManager**: 提供针对 `rooms` 数据的操作入口。内部实现了一个间隔 **15秒** 触发一次的 LibGDX `Timer.Task` 心跳循环（Heartbeat），通过发起 `PATCH` 更新 `last_ping` 字段防踢。
- **时区同步与 UTC Bug 修复 [Crucial]**: 修复了旧版直接将本地时间拼凑 `"Z"` 字符发往后端的严重时区错乱 Bug。现在的代码会在生成 ISO 8601 时间戳时，严格使用 `sdf.setTimeZone(TimeZone.getTimeZone("UTC"))` 将时间计算转化为真实的标准 UTC 时间，完美匹配 PostgreSQL `timestamptz` 的 `now()` 机制要求。

### 2.4 最佳实践界面层 (UI/Presentation Layer)

- **GScreen 体系的完全接入**: 彻底重构了最初原生的 `ScreenAdapter` 实现，完全接入引擎专有的 `GScreen` 体系。
- **架构改进细则**:
  - 移除了手动的 `Stage stage = new Stage()` 实例，转而使用 `GScreen` 体系提供的自动被管理的 `protected NeonStage stage` 属性。
  - 移除了渲染方法中手动的 `Gdx.gl.glClearColor` 和 `Gdx.gl.glClear`。
  - 移除了 `resize` 和 `dispose` 中重复处置代码。
  - 标准化了生命周期钩子，例如确保首先调用父类的 `super.show()` 和 `super.render(delta)` 来自动化环境托管。
- **测试选单集成**: 圆满将 `SupabaseLobbyScreen` 挂载于主测试屏幕 `TestSelectionScreen` 的注册字典中，开发者现可直接在游戏启动时一键进入“Netcode坦克大厅(Supabase)”。

## 3. 安全与规范更新

- **认证规范替换**: 为了顺应新版的 Supabase 架构准则，将网络连接凭证的变量名由过时的 `anon-key` 标记为 `legacy`，正式更名为 `publishable-key`。并在此次重构中通过唯一的 `SupabaseConfig` 去提供服务，明确了密钥的权限边界。
