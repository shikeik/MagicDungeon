# v0.7.3 同机联调修复与大厅体验优化报表

> 生成日期: 2026-03-02  
> 项目: MagicDungeon2 — GDEngine Netcode 模块  
> 版本: v0.7.3  
> 前置版本: v0.7.2 (Realtime Presence 云大厅重构)

---

## 一、概述

v0.7.2 完成了 Presence 云大厅的协议层和 UI 重构，但在实际双端联调中暴露三个问题：

1. **同机/同局域网连接失败**: 房主发布公网 IP，同一台机器的客户端尝试连接公网 IP 失败（无 NAT 回环）。
2. **日志格式不统一**: `PhoenixChannel` / `PresenceLobbyManager` 直接使用 `Gdx.app.log()`，输出没有时间戳和级别信息，与项目统一的 `DLog` 体系不一致。
3. **房间列表缺少地址信息**: UI 只显示房间名、人数、状态，看不到实际 IP:Port，调试不直观。
4. **无自定义 IP 能力**: 使用 Sakura-Frp 等端口转发工具时，代理 IP 不等于本机 IP，房主无法指定自己的连接地址。

本版本一次性修复以上所有问题，并消除 SLF4J 红色警告日志。

---

## 二、问题分析与方案

### 2.1 同机/同 LAN 连接失败

**根因分析**: 

```
                  ┌── checkip.amazonaws.com
                  │   返回公网 IP: 203.168.242.205
                  ▼
  实例A (Host) ──────► 发布房间 hostIp=203.168.242.205, hostPort=20001
                          ↑
  实例B (Client) ────► 读取 hostIp → connect(203.168.242.205:20001)
                          → UDP 发出 → 路由器/NAT 不支持回环 → 超时
```

路由器通常不支持 NAT Hairpin（回环），即同一内网机器无法通过公网 IP 访问自己。VPN 场景更复杂——`getLocalIP()` 可能返回 VPN 隧道适配器 IP 而非真正的 LAN IP。

**解决方案**: 发布双 IP（公网 + 局域网），加入时三级 IP 智能选择：

```
加入房间时:
  if (我的公网IP == 房主公网IP):
      if (我的局域网IP == 房主局域网IP):
          → 同一台机器，使用 127.0.0.1 (回环)
      else:
          → 同一局域网，使用 房主局域网IP
  else:
      → 不同网络，使用 房主公网IP
```

### 2.2 日志格式不统一

**根因**: `PhoenixChannel` 和 `PresenceLobbyManager` 在 v0.7.2 实装时直接使用了 `Gdx.app.log(TAG, msg)` / `Gdx.app.error(TAG, msg)` / `Gdx.app.debug(TAG, msg)` 原生 API，该 API 不带时间戳和级别标签。

**对比**:
```
原生 Gdx.app.log 输出:
  [PresenceLobby] Presence 全量同步完成，共 1 个房间

DLog 统一输出:
  [15:22:13:408] [DEBUG] [PresenceLobby] Presence 全量同步完成，共 1 个房间
```

**方案**: 将所有 `Gdx.app.log/error/debug` 调用替换为 `DLog.logT/logErrT` 系列。

### 2.3 房间列表缺少地址信息

房间行只有 `房间名 | 人数 | 状态 | [加入]`，调试时无法确认 IP 和端口是否正确。

**方案**: 在房间列表中新增 "地址" 列，显示格式 `ip:port`（优先显示客户端实际要连接的 IP）。

### 2.4 自定义 IP (Frp/Proxy 支持)

端口转发工具 (如 Sakura-Frp) 提供的代理地址不等于本机 IP，房主需要手动指定。

**方案**: 右侧操作面板增加 "主机地址(可选)" 输入框，空值 = 自动检测公网 IP，填入值 = 使用自定义 IP。

## 四、核心实现细节

### 4.1 三级 IP 智能选择

```java
// SupabaseLobbyScreen.joinRoom()
String connectIp = room.hostIp;  // 默认: 公网 IP

if (myPublicIp != null && myPublicIp.equals(room.hostIp)) {
    // 同公网 IP → 检测是否同机
    if (myLocalIp != null && myLocalIp.equals(room.localIp)) {
        connectIp = "127.0.0.1";  // 同机: 回环地址
    } else if (room.localIp != null && !room.localIp.isEmpty()) {
        connectIp = room.localIp; // 同 LAN: 局域网 IP
    }
}
```

### 4.2 自定义 IP 建房流程

```
用户输入 "主机地址(可选)" = "frp.example.com"
  → createAndPublishRoom() 检测 hostIpField 非空
  → 跳过 PublicIPResolver，直接使用用户输入的地址
  → publishRoom(roomName, "frp.example.com", localIp, port, ...)
  → 其他玩家看到房间地址为 frp.example.com:20001
```
